## Commands

### Tests (BATS — unit tests)

```bash
# Run all unit tests
cd tests && bats unit/

# Run a single test file
bats tests/unit/test_validation.bats
bats tests/unit/test_logger.bats
bats tests/unit/test_os_detection.bats

# Run integration tests (require Docker + deployed container)
bats tests/integration/test_user_workflow.bats
```

### Build Docker image

```bash
# Local build
docker build -f docker/familytraffic/Dockerfile docker/familytraffic/ -t familytraffic:local

# With custom mtg version
docker build --build-arg MTG_VERSION=2.2.3 --build-arg MTG_ARCH=amd64 \
  -f docker/familytraffic/Dockerfile docker/familytraffic/
```

### Trigger CI/CD

CI/CD (GitHub Actions) triggers only on `VERSION` file changes in the `test` branch. Bump `VERSION` to start a build. Branch `master` is the main branch (for PRs from `test`).

## Architecture

### Single-container design (v5.33)

One Docker container `familytraffic` runs four processes under **supervisord** (PID 1):

| Process | Role |
|---|---|
| xray | VLESS Reality (port 8443), Tier 2 transports (8444–8446), SOCKS5 (10800), HTTP proxy (18118) |
| nginx | SNI routing on port 443, TLS termination on 1080/8118, cloak-port 4443 (loopback-only) |
| certbot-cron | Auto-renews Let's Encrypt every 12 h |
| mtg | MTProxy (Fake TLS, port 2053) — optional, enabled via `familytraffic-mtproxy setup` |

`network_mode: host` — container shares the host network stack. No Docker DNS, no inter-container networking.

### Key paths on the host (mounted into container)

| Path | Purpose |
|---|---|
| `/opt/familytraffic/config/nginx/nginx.conf` | nginx config (generated by `lib/nginx_stream_generator.sh`) |
| `/opt/familytraffic/config/xray_config.json` | xray config (static) |
| `/opt/familytraffic/data/users.json` | Users DB (live-reloadable via `SIGHUP` to xray) |
| `/opt/familytraffic/config/mtproxy/mtg.toml` | mtg v2 config |
| `/opt/familytraffic/config/mtproxy/secrets.json` | MTProxy secrets |
| `/opt/familytraffic/config/supervisord.d/mtg.conf` | Enables mtg autostart; created by `mtproxy setup`, removed by `mtproxy disable` |
| `/opt/familytraffic/.env` | Runtime env vars: `GHCR_IMAGE`, `FT_IMAGE_TAG`, `DOMAIN` |

### Codebase layout

```
scripts/            — CLI entry points (installed to $PATH)
  familytraffic               Main CLI (user/transport/proxy management, service ops)
  familytraffic-mtproxy       MTProxy CLI (setup/start/stop/secrets)
  familytraffic-external-proxy  Upstream proxy management
  familytraffic-cert-renew    Certificate renewal wrapper

  mtproxy_manager.sh          mtg v2 lifecycle, supervisord integration, UFW
  mtproxy_secret_manager.sh   Secret generation/validation (ee-format)
  nginx_stream_generator.sh   Generates nginx.conf for SNI routing
  user_management.sh          VLESS user CRUD, UUID generation
  transport_manager.sh        Tier 2 transport enable/disable (ws/xhttp/grpc)
  external_proxy_manager.sh   Per-user upstream proxy chains
  validation.sh               Input validation (usernames, domains, etc.)
  ...

docker/familytraffic/
  Dockerfile        3-stage: xray-src → mtg-src → final (nginx:alpine + supervisor + certbot + xray + mtg)
  supervisord.conf  Base supervisord config (xray, nginx, certbot-cron)

```

### MTProxy (mtg v2) — critical details

- **Secret format:** `ee` + 32 hex chars (random key) + hex-encoded fake-domain. Length varies by domain.
- **Regex for validation:** `^ee[0-9a-fA-F]{34,}$`
- **mtg.toml structure:** `prefer-ip` must be at **top level** (not under `[network]`). DNS uses `udp://8.8.8.8` under `[network]`.
- **Port 4443 (cloak-port):** nginx listens only on loopback — never open in UFW.
- **Fake-domain flag:** `--fake-domain` (not `--domain`) in `familytraffic-mtproxy setup`.

### Traffic flow

```
Client:443  → nginx (ssl_preread SNI)
               ├─ Reality SNI  → 127.0.0.1:8443 (xray VLESS Reality)
               └─ Tier 2 SNI  → port 8448 → ws/xhttp/grpc inbounds
Client:1080 → nginx TLS termination → 127.0.0.1:10800 (SOCKS5)
Client:8118 → nginx TLS termination → 127.0.0.1:18118 (HTTP proxy)
Client:2053 → mtg (MTProxy Fake TLS) → Telegram DCs
Client:4443 → nginx (LE-cert, active-probing protection, loopback only)
```

### Versioning

The `VERSION` file (semver, e.g. `1.1.4`) is the single source of truth for CI/CD. Bumping it triggers a GitHub Actions build that pushes `:latest` + `:<version>` to GHCR.
